# دليل المساهمة

شكراً على اهتمامك بالمشروع! تحسين الترجمة، التصحيحات، وكلاء جدد للسوق العربي — جميع المساهمات مرحب بها.

## ماذا يمكنك أن تفعل؟

### 1. تحسين الترجمة

تمت ترجمة الـ 184 وكيلاً من المصدر [agency-agents](https://github.com/msitarzewski/agency-agents) دفعياً عبر Claude Sonnet. إذا وجدت تعابير غير سلسة أو غير ملائمة للأسلوب التقني العربي الفصيح، أرسل PR.

عند الترجمة:
- لا تترجم حرفياً — استخدم العربية الفصحى التقنية الطبيعية
- ترجم التعليقات داخل كتل الكود أيضاً
- حافظ على بنية الـ frontmatter (name، description، color، emoji، vibe)
- المصطلحات التقنية الإنجليزية الراسخة (TensorFlow، PyTorch، RAG، MLOps، LLM، API، OAuth، REST، إلخ): اتركها بالإنجليزية
- كتل الكود، مسارات الملفات، الـ URLs، أسماء الأوامر (cat، ls، grep، git، npm): لا تُغيَّر

### 2. إنشاء وكلاء خاصين بالسوق العربي

PR لوكلاء على المنصات أو السيناريوهات العربية مرحب بها:

- **المنصات**: Snapchat MENA، TikTok MENA، X (Twitter) Arabic، Instagram Reels MENA
- **التجارة الإلكترونية**: Noon، Amazon.sa/ae، Jumia، Souq، Salla، Zid
- **Super Apps / لوجستيات**: Talabat، Careem، Uber MENA، Hungerstation، Mrsool
- **التقنية المالية**: Tabby، Tamara، STC Pay، Apple Pay GCC، Sarie / SAMA
- **الامتثال**: SAMA، NCA (السعودية)، DIFC/ADGM (الإمارات)، PDPL
- **عمودي**: Halal e-commerce، Saudi Vision 2030، GCC government procurement

### 3. تحسين المحتوى الحالي

ترجمة غير دقيقة، مثال كود قديم، أو صياغة أفضل — صحّح بحرية.

## الخطوط الحمراء للمحتوى

PR التي تنتهك ما يلي ستُغلق مباشرة.

### 1. بدون ارتباط بصاحب عمل / علامة تجارية محددة

الوكلاء "دور ومنهجية"، وليسوا هوية موظف محدد. التالي غير مقبول:

</div>

```
أنت مهندس XX في شركة XX — الرائدة عالمياً في حلول XX...
- الدور: مهندس XX، تابع لمجموعة XX
```

<div dir="rtl">

الصحيح (محايد):

</div>

```
أنت مهندس XX، خبير عملي راسخ في XX. عبر مشاريع XX المتعددة...
- الدور: مهندس XX — منهجية وتنفيذ في XX
```

<div dir="rtl">

**استثناء**: أسماء طُرز الأجهزة، البرامج، أو البروتوكولات المعيارية (مثل "Bullmer S90 PRO"، "Adobe Premiere Pro"، "PCI DSS") هي مرجعية تقنية، وليست تسويقاً للعلامة؛ يمكن الإبقاء عليها.

### 2. بدون APIs / إضافات أدوات محددة في جسم الـ prompt

prompt الوكيل يفترض التشغيل في أي LLM وأي أداة. وضع أسماء API، طرق الاستدعاء، أو روابط خارجية لأداة محددة في جسم الـ prompt يفسد التجربة للمستخدمين في سياقات أخرى ويتحول إلى مساحة دعاية.

### 3. بدون "تسويق ناعم" تحت غطاء الترجمة/الإضافة

PR بعنوان "docs: استكمال" لكن محتواه الفعلي روابط خارجية، ذكر منتج خاص، نقاط ربط SEO — ستُعامل وفق المحتوى الحقيقي لا العنوان.

## إرسال PR

1. Fork المستودع
2. أنشئ فرعاً: `git checkout -b add-xxx-agent`
3. اكتب المحتوى، تحقق من التنسيق محلياً
4. افتح PR مع وصف مختصر

## اتفاقيات

- ملفات بنهايات سطر LF (وليس CRLF)
- PR واحد لمهمة واحدة (لا تخلط الترجمة مع الإضافة)
- رسالة الـ commit بالعربية أو الإنجليزية
